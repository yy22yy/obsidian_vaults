# 一、个人博客搭建的核心概念图

先建立一个完整的心智模型，然后每一层才有意义：

```
你写的内容（Markdown/Obsidian）
        ↓  [静态站点生成器，例如 Hexo/Hugo]
    HTML/CSS/JS 静态文件
        ↓  [部署/托管]
    服务器 or CDN（GitHub Pages / Vercel / 云服务器）
        ↓  [域名解析]
    你的域名（yourname.com）
        ↓
    用户浏览器
```

这四层每一层都是独立的技术选择。


```text
一个经典的逻辑：
本地的obsidian或者文件或者语雀的想法
--->>
编译成浏览可以解读的页面--HTML 
--->>
服务器的作用：存储好这些html页面并且将他们分发出去？
```

# Markdown --> HTML :[[Hexo]]
```
obsidian 写 .md 文件
	 → 放进 hexo/source/_posts/ 
	 → hexo generate 生成静态文件 
	 → hexo deploy 推送到 GitHub Pages 
```

## 托管层是什么 托管层解决的问题是：

**你生成的 HTML 文件，放在哪里让全世界都能访问。

| **[[GitHub Pages]]** | 免费 | 低 | 个人博客首选，简单够用 | 
| **[[Vercel / Netlify]]** | 免费起步 | 中 | 更快的 CDN，支持自动部署 |
