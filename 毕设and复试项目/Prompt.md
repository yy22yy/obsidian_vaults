## Prompt是基于什么需求定义的

### Prompt出现之前的问题

GPT-3（2020）发现了一个惊人的现象：**模型大到一定程度，不需要微调，只需要在输入里描述任务，它就能完成。**

```
传统微调方式：
  准备标注数据 → 更新模型参数 → 部署新模型
  成本：高，周期：长，灵活性：低

GPT-3的新发现：
  直接告诉模型要做什么 → 它就做了
  成本：零，周期：即时，灵活性：极高
```

这个"告诉模型要做什么的输入文本"就是**Prompt**。

---

### Prompt的本质需求

Prompt是为了解决一个核心矛盾：

> **模型已经有能力了，但不知道你现在想让它做什么。**

```
预训练后的模型就像一个博学的人：
  他懂语言、懂知识、懂推理
  但他不知道你现在想聊什么

Prompt就是你开口说的第一句话：
  "帮我翻译以下内容"    → 激活翻译能力
  "请分析这段代码的bug" → 激活代码分析能力
  "用小红书风格改写"    → 激活风格迁移能力
```

**Prompt不是在教模型新能力，而是在激活和引导它已有的能力。**

---

### Prompt的三种形态演变

```
Zero-shot Prompt（零样本）：
  只描述任务，不给例子
  "判断以下评论的情感：'这电影太棒了'"
  → 依赖模型的通用理解能力

Few-shot Prompt（少样本）：
  给几个例子，模型类比推断
  "正面：'很好看' 负面：'很难看' 请判断：'一般般'"
  → 通过例子临时定义任务格式

Instruction Prompt（指令）：
  详细的自然语言指令
  "你是一个专业编辑，请用正式语气改写以下文本，
   保留核心含义，长度控制在100字以内"
  → ChatGPT时代的主要交互方式
```
