## 编码器、解码器与BERT的历史意义

### 先理解原始Transformer的结构

原始Transformer（2017）是为**翻译任务**设计的，所以有两个部分：

```
"我爱北京"
     ↓
 ┌─────────┐
 │  编码器  │  ← 理解输入
 └─────────┘
     ↓
 ┌─────────┐
 │  解码器  │  ← 生成输出
 └─────────┘
     ↓
"I love Beijing"
```

---

### 编码器的作用：理解

编码器的任务是**读懂输入，建立深度语义表示**。

它可以同时看到句子的**所有词**，双向理解：

```
"苹果发布了手机"

编码器处理"发布"时：
← 可以看左边："苹果"
→ 可以看右边："手机"

最终："发布"的向量融合了左右两侧的完整语境
```

==编码器使用的是**双向自注意力**，每个词都能自由关注句子中任意位置。结果是每个词都得到一个**富含上下文的语义向量**。==

**核心特点：看全局，求理解，不生成新内容。**

---

### 解码器的作用：生成

解码器的任务是**逐词生成输出序列**。

但它有一个关键限制——**masked自注意力（掩码注意力）**：

```
生成"I love Beijing"的过程：

第一步：只能看 [开始符] → 生成 "I"
第二步：只能看 [开始符, I] → 生成 "love"
第三步：只能看 [开始符, I, love] → 生成 "Beijing"
```

==为什么要加这个限制？因为生成时**未来的词还不存在**，不能作弊偷看。这个限制叫做**因果掩码（causal mask）**。==

解码器还有**交叉注意力层**，专门用来"询问"编码器的输出：

```
解码器生成"love"时：
  自注意力：看已生成的 "I"
  交叉注意力：回头看编码器对"我爱北京"的理解
      ↓
  结合两者，生成最合适的下一个词
```

**核心特点：看局部（只看过去），逐步生成，不能看未来。**
