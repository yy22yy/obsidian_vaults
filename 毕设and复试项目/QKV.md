## 如何理解 Transformer 中的 QKV 设计

### 先从一个现实类比开始

想象你在用**搜索引擎**：

```
你输入："北京好吃的餐厅"         ← Query（你的需求）

数据库里每个网页都有标签：
  网页A："北京美食推荐"          ← Key（网页的自我描述）
  网页B："上海旅游攻略"          ← Key
  网页C："北京米其林餐厅评测"    ← Key

匹配后，你得到的实际内容：
  网页A的正文                    ← Value（真实信息）
  网页C的正文                    ← Value
```

- Query:需求
- Key：描述，别人声称是什么
- Value：真实信息

---

### 为什么一个词需要三个向量？

这是QKV设计最反直觉的地方。

同一个词，根据**它在做什么**，需要不同的表示：

```
"猫追了狗"中的"猫"：

作为Query时："我想知道谁是我的动作对象？"  → Q_猫
作为Key时：  "我是一个动物，是句子主语"    → K_猫  
作为Value时："我贡献的语义信息是..."       → V_猫
```

如果三个角色用同一个向量，就相当于：

- 你搜索时用的关键词
- 你网页的标签
- 你网页的内容

**三者完全相同**——这会让模型失去灵活性，无法分别优化"如何提问"和"如何被找到"。

---

### 三个向量是怎么来的

通过三个**独立的权重矩阵**线性变换：

```
词的嵌入向量 x（比如512维）
         │
    ┌────┼────┐
    Wq   Wk   Wv     ← 三个矩阵，形状各为 512×64
    │    │    │         训练过程中自动学习
    ↓    ↓    ↓
    Q    K    V        ← 每个64维
```

关键点：**Wq、Wk、Wv 是训练出来的**，模型自己学会了：

- 什么样的"提问方式"最有效（Wq）
- 什么样的"自我描述"最容易被匹配（Wk）
- 什么样的"贡献内容"最有价值（Wv）

---

### 完整计算流程

以句子 **"猫追了狗"** 为例，计算"追"这个词的新表示：

**Step 1：生成所有词的QKV**

```
         Q          K          V
猫  → [0.2,0.9]  [0.8,0.1]  [0.5,0.3]
追  → [0.7,0.4]  [0.3,0.6]  [0.2,0.8]
了  → [0.1,0.2]  [0.1,0.1]  [0.1,0.1]
狗  → [0.6,0.8]  [0.7,0.4]  [0.4,0.6]
```

**Step 2："追"的Q与所有词的K做点积**

```
"追"想问：谁和我的动作最相关？

查询相关性利用 Q*K ？

Score(追→猫) = Q_追 · K_猫 = 0.7×0.8 + 0.4×0.1 = 0.60  ← 主语，高分
Score(追→追) = Q_追 · K_追 = 0.7×0.3 + 0.4×0.6 = 0.45
Score(追→了) = Q_追 · K_了 = 0.7×0.1 + 0.4×0.1 = 0.11  ← 助词，低分
Score(追→狗) = Q_追 · K_狗 = 0.7×0.7 + 0.4×0.4 = 0.65  ← 宾语，高分
```

**Step 3：缩放后Softmax**

```
为什么要除以 √dk ？
→ 维度越高，点积值越大，Softmax会变得极端（接近one-hot）
→ 梯度会消失，训练不稳定
→ 除以 √dk 把数值拉回合理范围

原始分数：[0.60, 0.45, 0.11, 0.65]
  ÷ √2  ：[0.42, 0.32, 0.08, 0.46]
Softmax ：[0.29, 0.24, 0.17, 0.30]  ← 注意力权重，加和=1
```

**Step 4：加权求和Value**

```
新的"追" = 0.29×V_猫 + 0.24×V_追 + 0.17×V_了 + 0.30×V_狗

         = 0.29×[0.5,0.3] + 0.24×[0.2,0.8] + 0.17×[0.1,0.1] + 0.30×[0.4,0.6]

         = [0.34, 0.54]
```

这个新向量不再是孤立的"追"，而是**融合了"猫"（主语）和"狗"（宾语）语义的"追"**。

---

### Q和K分离的深层原因

这是设计中最精妙的地方：

```
对称 vs 不对称的关系

如果Q=K：
  "猫"关注"狗"的程度 = "狗"关注"猫"的程度（强制对称）

Q≠K时：
  "猫"（作为主语）关注"狗" 的程度
  ≠
  "狗"（作为宾语）关注"猫" 的程度
```

现实语言中，关系本来就是不对称的：

- "动词"对"宾语"的依赖 ≠ "宾语"对"动词"的依赖
- "代词"对"先行词"的依赖 >> "先行词"对"代词"的依赖

==**Q和K分离，让模型可以学习这种天然的不对称性。**==

---

### Multi-Head：为什么要多组QKV？

单组QKV只能学习一种关注模式，但语言有多个维度：

```
同一句"猫追了狗"，不同的头关注不同的事：

Head 1（句法头）：  关注主谓宾结构
  "追" → 高度关注"猫"（主语）和"狗"（宾语）

Head 2（语义头）：  关注语义相似性
  "猫" → 高度关注"狗"（同为动物）

Head 3（位置头）：  关注相邻关系
  每个词 → 高度关注相邻的词
```

多个头并行计算，最后拼接：

```
[Head1输出 | Head2输出 | Head3输出 | ...]
              ↓  线性变换
         最终输出向量
```

**每个头用独立的Wq、Wk、Wv，自由地学习不同维度的语言规律。**

---

### 一句话总结QKV的本质

> Q决定**"我想要什么"**，K决定**"我能被什么找到"**，V决定**"被找到后我贡献什么"**。
> 
> 三者分离，让每个词同时成为一个**灵活的提问者、被检索者和信息贡献者**，从而实现无损、直接、不对称的语义交互。
