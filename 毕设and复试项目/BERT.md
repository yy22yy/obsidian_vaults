### [[BERT]]为什么移除解码器？

==因为BERT的目标**根本不是生成文本**，而是做最好的语言理解工具==

解码器的因果掩码对BERT来说是一种**主动的残缺**：

```
如果BERT保留解码器，处理"银行"时：

"他去银行取钱"
  他  去  银行  取钱
  ✅  ✅   ✅   ❌  ← 解码器看不到右边

但理解"银行"到底需要右边的"取钱"！
```

==BERT只保留编码器，让每个词**自由地双向关注所有词**，从而得到最充分的语义理解==

### BERT在历史上解决了什么问题

**2018年之前的困境：每个任务都要从头训练**

```
情感分析任务  → 训练一个模型
问答任务     → 重新训练一个模型
命名实体识别  → 再训练一个模型
文本分类     → 又训练一个模型
```

每个模型都需要大量**该任务的标注数据**，而标注数据既昂贵又稀缺。

**BERT的解决方案：预训练 + 微调**

```
第一阶段：预训练（在海量无标注文本上）
Wikipedia + BookCorpus（30亿词）
           ↓
    学习通用语言理解能力
           ↓
      BERT基础模型

第二阶段：微调（在少量任务数据上）
    BERT基础模型
    + 少量标注数据
           ↓
    ┌──────────────┐
    │情感分析模型   │
    │问答模型      │
    │文本分类模型  │
    └──────────────┘
```

### BERT的两个预训练任务

BERT用了两个巧妙的**自监督任务**，不需要人工标注：

**① 掩码语言模型（MLM）**

```
原句：  "他去银行取钱"
输入：  "他去[MASK]取钱"
任务：  预测[MASK]是"银行"

← 必须同时理解左边"他去"和右边"取钱"才能答对
← 这就逼迫模型学习双向语境
```

**② 下一句预测（NSP）**

```
句子A："他去银行取钱"
句子B："柜员态度很好"  → 判断：是连续的两句话吗？✅

句子A："他去银行取钱"
句子B："今天天气真好"  → 判断：是连续的两句话吗？❌

← 逼迫模型学习句子级别的语义关系
```

### BERT的历史意义

```
Word2Vec（2013）
  └→ 静态词向量，词有了语义，但语境是固定的

Transformer（2017）
  └→ 动态语境，但主要用于翻译生成任务

BERT（2018）
  └→ 把Transformer的理解能力工业化：
     "一个模型，解决所有NLP任务"
```

BERT发布后，在11项NLP基准测试上**同时刷新了最高分**，这在当时是震惊整个领域的成就。

更深远的意义是：**它确立了"[[预训练]]+[[微调]]"这个范式**，这个思路直接延续到了今天所有的大模型——GPT、Claude、Gemini，本质上都是这个框架的延伸和扩展。
