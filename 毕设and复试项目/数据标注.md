## 一、标注 vs 未标注数据

### 最直观的对比

```
未标注数据（原始文本）：
"这部电影的剧情非常精彩，演员表演也很到位"

标注数据（人工加上了答案）：
"这部电影的剧情非常精彩，演员表演也很到位" → 【正面情感】
```

**标注就是人类告诉模型"正确答案是什么"。**

---

### 标注数据的本质

标注数据本质上是在做一件事：**把人类的判断转化为训练信号**。

不同任务需要不同类型的标注：

```
情感分析标注：
  文本 → 正面/负面/中性

命名实体识别标注：
  "[苹果]公司发布了新[iPhone]"
   └─企业─┘         └─产品─┘

机器翻译标注：
  "我爱北京" → "I love Beijing"

问答标注：
  问题："牛顿在哪里发现了万有引力？"
  文章："...牛顿在英国乡下看到苹果落地..."
  答案：起始位置第X个字，结束位置第Y个字
```

每一条标注都需要**人类专家花时间判断**，这就是为什么标注数据稀缺且昂贵。

---

### 为什么未标注数据也能训练---自监督学习

这是预训练最聪明的地方——**从文本本身构造监督信号**：

```
BERT的掩码任务：
原文（未标注）："牛顿在英国提出了万有引力定律"
自动构造：     "牛顿在[MASK]提出了万有引力定律"
自动标注：      答案 = "英国"  ← 不需要人，文本自己提供答案

GPT的预测任务：
原文（未标注）："今天天气很好"
自动构造：     "今天天气很" → 预测下一个词
自动标注：      答案 = "好"  ← 同样不需要人
```

**未标注数据通过这种方式变成了"自带答案的题目"，** 这叫做**自监督学习**，是预训练能用海量数据的根本原因。

---

### 两者的使用分工

```
训练阶段          数据类型        规模
─────────────────────────────────────
预训练           未标注数据      数十亿句
                （自动构造标签）

微调             标注数据        几百~几万句
                （人工标注）

现代大模型的
指令微调(RLHF)   标注数据        几万~几十万条
                （人工写答案+排序）
```
