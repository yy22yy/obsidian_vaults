# 一、对抗样本攻击 vs 数据投毒攻击

| 维度     | 对抗样本攻击              | 数据投毒攻击             |
| ------ | ------------------- | ------------------ |
| 攻击阶段   | **推理阶段（Inference）** | **训练阶段（Training）** |
| 攻击对象   | 输入样本                | 训练数据               |
| 是否修改模型 | 不修改                 | 间接影响模型             |
| 攻击方式   | 添加微小扰动              | 注入恶意训练数据           |
| 影响范围   | 通常针对单个样本            | 可能影响整个模型           |

---

## 1 对抗样本攻击（Adversarial Example）

核心思想：

> 在输入数据上加入微小扰动，使模型产生错误预测。

特点：
- 人类几乎看不出差异
- 模型预测完全错误

常见攻击算法：
- Fast Gradient Sign Method
- Projected Gradient Descent

例子：
```
原始图片：熊猫
模型输出：熊猫

加入微扰

人类：还是熊猫
模型：长臂猿
```

**关键点**

攻击发生在：

```
模型已经训练好之后
```

攻击者只控制 **输入数据**。

---

## 2 数据投毒攻击（Data Poisoning）

核心思想：

> 在训练数据中加入恶意样本，让模型学习错误模式。

攻击过程：

```
攻击者 → 修改训练集 → 模型训练 → 模型产生错误行为
```

例子：

自动驾驶训练集：

```
STOP sign
```

攻击者加入：

```
带贴纸的 STOP sign → 标签为 speed limit
```

训练后：

```
看到贴纸 → 识别错误
```

这种攻击属于：

```
训练阶段攻击
```

---

## 3 更本质的区别

本质不是 **偏差 vs 对错**。

更准确的理解：

|类型|本质|
|---|---|
|对抗样本|利用模型 **决策边界脆弱性**|
|数据投毒|改变 **训练数据分布**|
**对抗样本**

```
攻击输入
```

**数据投毒**

```
攻击学习过程
```
