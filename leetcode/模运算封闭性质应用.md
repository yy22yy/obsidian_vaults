# 模运算封闭性与应用

#数学 #模运算 #数论 #算法 #密码学

## 基本定义

对任意整数 $a$ 和正整数 $k$，带余除法保证唯一分解：

$$a = k \cdot q + r, \quad 0 \leq r < k$$

**整除就是余数为零的特殊情况：**

$$k \mid a \iff a \mod k = 0$$

**同余关系** $a \equiv b \pmod{k}$ 等价于 $k \mid (a - b)$，即 $a$ 和 $b$ 被 $k$ 除余数相同。

---

## 封闭性定义

集合 $S$ 对运算 $\circ$ 封闭，是指：

$$a, b \in S \implies a \circ b \in S$$

模运算的余数集合是 ${0, 1, \ldots, k-1}$，封闭性要求在这个集合上做运算后结果仍落在集合内。

---

## 三条封闭性质

### 加法封闭

$$((a \bmod k) + (b \bmod k)) \bmod k = (a + b) \bmod k$$

> [!note] 证明思路 设 $a = kq_1 + r_1$，$b = kq_2 + r_2$，则 $$a + b = k(q_1 + q_2) + (r_1 + r_2)$$ $r_1 + r_2$ 再对 $k$ 取模，余数与 $(a+b) \bmod k$ 一致。

### 乘法封闭

$$((a \bmod k) \times (b \bmod k)) \bmod k = (a \times b) \bmod k$$

> [!note] 证明思路 展开 $a \times b = k^2 q_1 q_2 + k(q_1 r_2 + q_2 r_1) + r_1 r_2$，前两项均被 $k$ 整除，故结果只取决于 $r_1 \times r_2$。

### 减法封闭

$$((a \bmod k) - (b \bmod k) + k) \bmod k = (a - b) \bmod k$$

> [!warning] 加 $k$ 防负数 加 $k$ 是为了防止出现负余数，保证结果落在 $[0, k)$ 内。

---

## 为什么需要"再次取模"

加法后 $r_1 + r_2 < 2k$，可能 $\geq k$，不是合法余数，必须再压回 $[0, k)$。

**示例：** $k=7,\ a=10,\ b=11$

```
a mod 7 = 3
b mod 7 = 4
3 + 4   = 7   ← 已经 >= k
7 mod 7 = 0   ← 必须再取模

验证：(10 + 11) mod 7 = 21 mod 7 = 0 ✓
```

这与乘法分配律不同——分配律是实数域上的恒等式，而模运算每次运算后都要将结果"投影"回 $[0, k)$，即环形数轴上的位置。

---

## 除法为何不封闭

在 $\mathbb{Z}/k\mathbb{Z}$ 中，$a$ 有乘法逆元当且仅当 $\gcd(a, k) = 1$。

**反例：** $k=6$，元素 $2$ 没有逆元

$$2 \times {1,2,3,4,5} \bmod 6 \in {2, 4, 0, 2, 4}$$

永远无法得到 $1$，故除以 $2$ 在 $\mathbb{Z}/6\mathbb{Z}$ 中无意义。

> [!tip] 当 $k$ 为素数时 $\mathbb{Z}/k\mathbb{Z}$ 中每个非零元素都有逆元，构成**域**，除法对所有非零元封闭。这是算法竞赛模数常取质数（如 $10^9+7$）的原因。

---

## 代数结构

${0, 1, \ldots, k-1}$ 在加法和乘法下构成**环**，记作 $\mathbb{Z}/k\mathbb{Z}$。

三条封闭性的统一表述：**模 $k$ 的余数只取决于等价类，不取决于代表元的选取。**

即若 $a \equiv a' \pmod{k}$，$b \equiv b' \pmod{k}$，则：

$$a \pm b \equiv a' \pm b' \pmod{k}$$ $$a \times b \equiv a' \times b' \pmod{k}$$

---

## 应用一：Pascal 递推与组合数取模

### 问题背景

> [[P2822 组合数问题]]：判断 $C(i,j)$ 是否能被 $k$ 整除。 $C(2000, 1000)$ 远超 `long long` 范围，无法直接计算。

### Pascal 递推原理

对第 $n$ 号元素做二分讨论：

- **不选**：从 $n-1$ 个中选 $r$ 个 → $C(n-1, r)$
- **选**：剩余名额从 $n-1$ 中选 $r-1$ 个 → $C(n-1, r-1)$

$$C(n, r) = C(n-1, r-1) + C(n-1, r)$$

### 封闭性的作用

>递推只含加法，加法对模运算封闭，故：

$$C(n,r) \bmod k = \bigl(C(n-1,r-1) \bmod k + C(n-1,r) \bmod k\bigr) \bmod k$$

每步中间值始终在 $[0, k-1]$，**永远不会溢出**。

### 为什么不用公式 $C(n,r) = \frac{n!}{r!(n-r)!}$

>除法不封闭，中间步骤的除法打破模运算相容性。这是递推比公式更适合此题的代数本质。

### 代码实现

```cpp
// 分开维护：余数 和 是否 >= k
int rem[MAXN][MAXN];  // C(i,j) % k
int val[MAXN][MAXN];  // min(C(i,j), k+1)，判断 >= k

for (int i = 0; i < MAXN; i++) {
    val[i][0] = 1; rem[i][0] = 1 % k;
    for (int j = 1; j <= i; j++) {
        val[i][j] = min(val[i-1][j-1] + val[i-1][j], k + 1);
        rem[i][j] = (rem[i-1][j-1] + rem[i-1][j]) % k;
    }
}
// 条件：rem[i][j] == 0 && val[i][j] >= k
```

> [!tip] val 截断到 $k+1$ 的原因 加法不溢出（最大 $2(k+1)$ 再 min 回去），且足以判断 $\geq k$，与 rem 的计算互不干扰。

## 应用二：RSA 加密

### 基本框架

|操作|公式|
|---|---|
|密钥生成|选 $e$ 使 $\gcd(e, \varphi(n))=1$，求 $d$ 使 $ed \equiv 1 \pmod{\varphi(n)}$|
|加密|$c = m^e \bmod n$|
|解密|$m = c^d \bmod n$|

其中 $n = p \times q$，$p, q$ 为大素数。

### 封闭性的作用

**大数幂运算（快速幂）：** $m^e$ 直接计算不可存储，利用乘法封闭性逐步取模：

$$m^e \bmod n = \left(m^2 \bmod n\right)^{e/2} \bmod n$$

每步平方后立刻取模，中间结果始终 $< n$。

**解密正确性：**

$$c^d \bmod n = (m^e)^d \bmod n = m^{ed} \bmod n$$

由欧拉定理 $m^{\varphi(n)} \equiv 1 \pmod{n}$，且 $ed \equiv 1 \pmod{\varphi(n)}$，故 $m^{ed} \equiv m \pmod{n}$。

整个推导链每一步都依赖乘法封闭性保证等价类运算自洽。

### 安全性与封闭性的关系

封闭性使模运算成为**单向阀**：

- **正向（加密）：** 封闭性保证高效计算
- **逆向（破解）：** 需分解 $n = p \times q$，计算上不可行

若没有封闭性，运算结果逃出等价类，整个体系无法构造。

---

## 两个应用的对比

> [!example] 组合数问题
> - **利用的封闭性：** 加法封闭
> - **目的：** 防止中间值溢出
> - **不能用除法的原因：** 破坏模运算相容性
> - **素数的作用：** 无直接关系

> [!example] RSA
> - **利用的封闭性：** 乘法封闭
> - **目的：** 使大数幂运算可行
> - **不能用除法的原因：** 无逆元时无意义
> - **素数的作用：** 保证逆元存在，构成域



> [!summary] 共同本质 把无限大的整数运算压缩到有限的等价类上，**封闭性保证这种压缩是自洽的**。
