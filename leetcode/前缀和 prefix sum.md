# 前缀和（Prefix Sum）算法 - 核心原理与思想

## 1. 最核心的一句话

**“把从起点到每个位置的累加和提前算好，之后任意区间的和都可以用两个前缀值相减得到”**

这就是前缀和存在的全部意义。

## 2. 为什么需要前缀和？

原始方式求区间和：
- 每次询问 [L,R] → 都要从 L 循环加到 R
- 单次 O(R-L+1)，多次询问 → O(n) ~ O(n²)

前缀和方式：
- 预处理 O(n)
- 每次区间查询 O(1)

**典型收益**：把 O(n) 的多次区间求和 → 变成 O(1)

    pre[i] = pre[i - 1] + a[i - 1];
}

// 查询 [L,R] 闭区间和
long long sum(int L, int R) {
    return pre[R + 1] - pre[L];
}
