## 树的节点定义方法

### 核心思想

**你需要在递归/遍历过程中，每个节点上能直接访问哪些信息？**

---
## 一、指针式 vs 下标式

### 指针式

```cpp
struct Node {
    int val;
    Node* left;
    Node* right;
    Node(int v) : val(v), left(nullptr), right(nullptr) {}
};
```

**适合**：动态建树，节点数不确定，LeetCode风格题目。  
**危险**：存在`vector<Node>`中时，扩容会导致指针失效。

### 下标式

```cpp
struct Node {
    int val;
    int left = -1;
    int right = -1;
};
vector<Node> arr(n + 1);  // 用下标访问 arr[u].left
```

**适合**：节点数已知，竞赛题，更安全。  
**原则**：只要节点数已知，优先用下标式。

### 指针式和下标式的区别

**指针式** — 节点散落在堆内存各处

```
root ──→ [val=1 | left=0xA | right=0xB]
                     ↓              ↓
             [val=2 | ...]   [val=3 | ...]

每个节点在堆内存的不同位置，指针存实际内存地址
```

**下标式** — 节点紧密排列在连续内存

```
arr[0] [arr[1]         ] [arr[2]    ] [arr[3]    ] ...
       [val=1 | left=2 ] [val=2|... ] [val=3|... ]
       [right=3        ]

"指针"就是数组下标，本质是偏移量
```

---

### 指针式的致命陷阱

> [!danger] vector 扩容导致野指针 把 `Node` 对象存进 `vector`，再取地址存为指针，扩容后指针失效！

cpp

```cpp
vector<Node> arr(n + 1);
arr[1].left = &arr[2];   
》》这里要使得等式的两边有意义，Node* left = 一个Node对象的内存地址
》》如果Node定义的是int left 那么 Node.left = 一个int对象
》 存了 arr[2] 的地址A
》 这里直接 = arr [2]是错误的理解；arr[2]是一个node对象，存储的应该是arr[2]的地址

---

arr.push_back(something); // vector 扩容，整块内存被移走！
arr[1].left;              // ❌ 野指针！原地址已失效
```

```
扩容前：arr 在地址 0x100，arr[1].left = 0x110
扩容后：arr 被移到 0x200，arr[2] 现在在 0x210
        但 arr[1].left 还是 0x110  ← 野指针
```

**下标式完全没有这个问题**，下标 `2` 永远指向 `arr[2]`，无论内存怎么移动。

*解释*
>指针式--==物理位置==；
>下标式--==相对位置==;
>

### 下标版（竞赛标准）

```cpp
struct Node{  
    int left;  
    int right;  
};  
  
vector<Node> tree;

访问：

tree[u].left  
tree[u].right
```
优点：

1. **不会因扩容失效**
2. **节省内存**
3. **cache 友好**
4. **序列化方便**
5. **调试简单**
---

### 指针式的正确用法

节点数未知时，用 `new` 逐个创建，**不放进 vector**：

```cpp
Node* newNode(int val) {
    return new Node(val);
}
Node* root = newNode(1);
root->left = newNode(2);  // new 出来的地址永远稳定 ✅
```

## 二、按题型分类

### 1. 只需要遍历/计算结构属性

> 求深度、宽度、节点数、路径等

```cpp
struct Node {
    int left = -1;
    int right = -1;
};
```

只需要知道孩子是谁，不需要其他信息。

---

### 2. 需要向上回溯

> 求LCA、求节点到根的路径、求距离

```cpp
struct Node {
    int left = -1;
    int right = -1;
    int parent = -1;  // 多一个父节点
};
```

或者不存父节点，用**额外数组**记录：

```cpp
vector<int> parent(n + 1);
vector<int> depth(n + 1);
```

---

### 3. 已知序列建树

> 已知前序+中序、后序+中序等

```cpp
// 不需要显式定义节点结构
// 直接用递归，传递序列的左右边界
int build(string pre, string post) { ... }

// 或者用下标范围，避免substr复制
int build(int preL, int preR, int inL, int inR) { ... }
```

---

### 4. 节点带权值

> 求带权路径、树形DP

```cpp
struct Node {
    int val;            // 节点权值
    int left = -1;
    int right = -1;
    int edgeWeight = 0; // 或者边权存在父节点上
};
```

边权通常存在**子节点上**（表示从父到我的边的权重）：

```cpp
struct Node {
    int left = -1;
    int right = -1;
    int weightToParent = 0;  // 到父节点的边权
};
```

---

### 5. 多叉树

> 题目没有限制二叉

```cpp
struct Node {
    int val;
    vector<int> children;  // 任意多个孩子
};
// 或者直接用邻接表
vector<vector<int>> children(n + 1);
```

---

### 6. 需要额外统计信息（树形DP）

> 求子树大小、重心、直径等

```cpp
struct Node {
    int left = -1;
    int right = -1;
    int size = 0;    // 子树节点数
    int dp = 0;      // DP值，按题意定义
};
```

或者完全分离，节点只存结构，DP值用额外数组：

```cpp
vector<int> sz(n + 1);   // 子树大小
vector<int> dp(n + 1);   // DP值
```

## 三、决策流程

```
题目给的是什么？
├── 给边 (u,v) 建树
│   ├── 保证u是父节点 → 直接建，存left/right
│   └── 无向边 → 邻接表建图，BFS/DFS确定父子关系
│
├── 给前序/中序/后序序列
│   └── 递归建树，传边界下标，不需要显式节点结构
│
└── 给括号/其他格式
    └── 按格式解析，动态建树用指针式

需要的操作是什么？
├── 只向下遍历 → 只存 left, right
├── 需要向上回溯 → 额外存 parent
├── 需要层序信息 → 额外存 depth
└── 需要统计子树信息 → 额外存 size/dp
```

## 四、通用模板

```cpp
// 竞赛中最常用的形式：结构与属性分离
struct Node {
    int left = -1;
    int right = -1;
};

vector<Node> arr(n + 1);  // 树结构
vector<int> parent(n + 1, -1);  // 需要时加
vector<int> depth(n + 1, 0);    // 需要时加
vector<int> sz(n + 1, 0);       // 需要时加
```

**核心原则**：节点结构只存**树的拓扑关系**，其他属性用**独立数组**按节点编号存储，保持结构清晰，按需添加。
