- 一个重要的事实：如果能够描述（递归地）一个树的root和左右子树的关系，那么这个树就可以被唯一描述出来

# 前中后序确定性差异
## 差异产生的本质

### 核心：中序遍历的作用是"定向"

```
中序: 左子树 | 根 | 右子树
```

中序的意义在于，根把序列**强制分成左右两侧**，左边一定是左子树，右边一定是右子树。这个"方向性"是刚性的。
由于知道了root ，方向确定，子树的长度也能确定

### 前序+后序缺少什么？

前序和后序都能找到根，但它们描述子树的方式是：

```
前序: 根 | 左子树 | 右子树   ← 左右子树紧密排列，靠"长度"区分
后序: 左子树 | 右子树 | 根   ← 左右子树紧密排列，靠"长度"区分
```

两者都依赖**长度**来划分左右子树，而长度信息来自于互相印证：  
前序的第二个元素（左子树根）在后序中的位置，决定了左子树的长度。

root（==左root1== 单位签订qwqko13）（==右root2==dwqdqwdq）
(1u01d0912hd1  ==左root2==)(都会0文化的1h ==右root1==) root
	>具体的确定方法是用 pre：左root1去找左root2；
	>然后左子树的长度就确定了
	
这里有一个问题，你按照左root1的语义进行理解时，左root2的语义就被确定了；
结果论来说，你用 后序确定前序 和  前序确定后序 的两种方法是一样的；
你初始的语义会产生不同的结果

### 歧义的产生

当左子树长度为 1，即某节点**只有一个孩子**时：

```
前序: a b ...
后序: ... b a
```

我们只知道 `b` 这个子树长度为1，但**无法判断 b 是 a 的左孩子还是右孩子**。

```
情况1:       情况2:
  a             a
 /               \
b                 b
```

这两棵树的前序和后序**完全相同**，但中序不同：

- 情况1中序：`b a`
- 情况2中序：`a b`

**根本原因：前序和后序只编码了"谁先谁后"的时间顺序，没有编码"谁在左谁在右"的空间方向。只有中序遍历天然携带了左右方向信息。**

# 树的构造
### 一句话总结

> 前序/后序 → 告诉你树的**层次结构和长度**  
> 中序 → 告诉你每个节点的**左右方向**
> 
> 缺少中序时，每个"独生子节点"都可以左右横跳，贡献一个 ×2 的自由度。
1. 这两个序列很容易确定根的位置；
2.  按照中序 left | root | right;找出左右子树
3. 然后可以根据左右子树各自的长度在 pre或者post中截取子字符串
4. 然后递归处理

==本质上还是要用到递归的思想==
	处理好当前层（或者理解为最上层）然后向下传递

## 1. 利用前序或者后序遍历取根操作
```cpp
char root;

pre:
root = string.begin();
root = string.back();

```


## 2. 用root找出中序中的位置
```cpp

int in_pos = inorder.find(root);
```

## 3.划分左右子树
```cpp
>中序序列的左右子树划分
string inLeft  = inorder.substr(0,in_pos);
string inRight = inorder.substr(in_pos+1);
>后序序列的左右子树划分
string poLeft  = postorder.substr(0,inLeft。size());//要根据中序遍历确定前面子树长度
string poRight = postorder.substr(in_pos+1);
```
